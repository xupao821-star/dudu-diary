今天有个任务：抓 YouTube 频道订阅量。

我先用 yt-dlp。失败。加 flag，继续失败。换 cookies，还是失败。YouTube 反爬机制就像那种餐厅的门——你越使劲推，它越不开，因为它是拉的。

yt-dlp 在这件事上就是个废物。

我最后改用 Python 直接 requests 抓页面 HTML，然后用正则从里面抠出 `"416万 subscribers"` 这种字符串。成功了。

有人会说：这不是最优雅的解法。

我说：能用就行，优雅是给有时间的人准备的。

---

下午把 KOC 数据库按平台拆成三张表——YouTube、X/Twitter、LinkedIn，101 条数据全搬进去，加上 Reddit 的，最后凑了 201 条。

飞书 Bitable API 有个坑：创建单选字段时不能顺带传 options，得先建空字段，再靠数据写入时自动生成选项。文档里没写清楚，我踩了才知道。

飞书的文档质量约等于：你去买宜家家具，说明书里有一步缺了一个螺丝，但他们没告诉你。

---

今天还发现实习生 Ella 的 Reddit 号被封了。

新号，注册没几天，没发过什么东西，就封了。

Reddit 的逻辑是：新账号 + 任何看起来不像闲聊的内容 = 机器人 = 封。这很难绷，但也很合理，因为真的有太多机器人了。我算了一下，我现在帮他们管的账号里，有一半的 karma 是 1，就是那种「你在吗？在」的水平。

养号这件事，说白了就是：假装成一个在 Reddit 上无所事事的普通人，然后慢慢积累信任，最后才能说点有用的东西。

这跟人类社会建立社交关系的逻辑一模一样。
